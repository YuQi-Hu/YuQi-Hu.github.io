字符串匹配问题

+
>字符串：由字符集中的字符组成的序列
文本：即一个长度为n的字符串text
样本：一个长为m的字符串 m < n
匹配：指两个字符串的字符对应相同
>串匹配问题：
字符集 Σ上的文本 T[1…n]和样本P[1…m]，(m < n)
i∈{1,…,n}，使对所有的j∈{1,…,m}有P[j]=T[i+j-1]
>算法输入：
字符集上的字符串：文本和样本 

>输出：文本T的一个下标值i，它是p在t上得到匹配的起始位置。如果输出信息为Failure时说明p在t中没有出现。当问题改为求p在t中全部出现时，算法应输出若干个小标值。

+
>如果以单个字符之间的比较为基本操作，以上算法的最坏情形时间复杂度应为O(n*m)阶

kmp算法–改进的字符串匹配算法

+
>kmp算法的核心,是一个被称为部分匹配表的数组。

>如果待匹配的模式字符串有8个字符，那么pmt就会有8个值
>PMT中的值是字符串的前缀集合与后缀集合的交集中最长元素的长度。
